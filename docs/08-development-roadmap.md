## 🔄 開發流程

### 每日流程

1. 晨會（可選）- 15 分鐘，同步進度
2. 開發 - 實作當日任務
3. Code Review - 提交前檢查
4. 測試 - 功能測試
5. 提交 - Git commit + push

### 分支策略

```
main          # 生產分支
  │
  ├── develop # 開發分支
  │     │
  │     ├── feature/user-auth      # 功能分支
  │     ├── feature/project-api
  │     └── ...
  │
  └── hotfix/xxx # 緊急修復
```

### 提交規範

| 類型 | 描述 |
|------|------|
| feat | 新功能 |
| fix | 修復 bug |
| docs | 文件更新 |
| style | 程式碼格式 |
| refactor | 重構 |
| test | 測試 |
| chore | 建構或輔助工具 |

範例：

```
feat(auth): 新增 Google 第三方登入
fix(task): 修復拖拽任務時的排序問題
docs(readme): 更新部署說明
```
